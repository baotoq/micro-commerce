  <action>
    Rewrite the bootstrap script to integrate ArgoCD installation and hand off resource management. The new flow:

    **Steps 1-2: KEEP UNCHANGED** (kind cluster creation, kubectl context)

    **Steps 3-4: KEEP UNCHANGED** (SealedSecrets controller install + wait)

    **Step 5: KEEP UNCHANGED** (create namespace)

    **Step 6: Seal dev secrets -- MODIFY the seal_secret calls.** Keep the seal_secret function and all 3 seal_secret calls (postgres, rabbitmq, keycloak) exactly as-is. After all 3 secrets are sealed, add explicit kubectl apply for each sealed-secret.yaml file:
    ```bash
    info "Applying sealed secrets..."
    kubectl apply -f "$SCRIPT_DIR/base/postgres/sealed-secret.yaml"
    kubectl apply -f "$SCRIPT_DIR/base/rabbitmq/sealed-secret.yaml"
    kubectl apply -f "$SCRIPT_DIR/base/keycloak/sealed-secret.yaml"
    ```
    WHY: sealed-secret.yaml is no longer referenced in base kustomization.yaml files (removed in Plan 01), so they must be applied explicitly. These must exist in the cluster before ArgoCD syncs the Deployments that reference the Secrets.

    **Step 7: REMOVE** the old `kubectl apply -k "$SCRIPT_DIR/base/"` line. ArgoCD will handle this now.

    **Step 8: REMOVE** the old infra pod wait commands. ArgoCD sync handles deployment.

    **NEW Step 7: Install ArgoCD v3.3.2**
    ```bash
    # --- Step 7: Install ArgoCD ---
    ARGOCD_VERSION="v3.3.2"
    info "Installing ArgoCD ${ARGOCD_VERSION}..."
    kubectl create namespace argocd --dry-run=client -o yaml | kubectl apply -f -
    kubectl apply -n argocd --server-side --force-conflicts \
      -f "https://raw.githubusercontent.com/argoproj/argo-cd/${ARGOCD_VERSION}/manifests/install.yaml"
    ```
    Use `--server-side --force-conflicts` because ArgoCD CRDs exceed 256KB annotation limit for client-side apply. Use idempotent namespace creation same as Step 5 pattern.

    **NEW Step 8: Wait for ArgoCD to be ready**
    ```bash
    info "Waiting for ArgoCD to be ready..."
    kubectl rollout status deployment argocd-server -n argocd --timeout=120s
    kubectl rollout status deployment argocd-repo-server -n argocd --timeout=120s
    kubectl rollout status deployment argocd-applicationset-controller -n argocd --timeout=120s
    ```

    **NEW Step 9: Configure ArgoCD for local access**
    ```bash
    info "Configuring ArgoCD for local access..."
    kubectl apply -f "$SCRIPT_DIR/argocd/argocd-cmd-params-cm.yaml"
    kubectl rollout restart deployment argocd-server -n argocd
    kubectl rollout status deployment argocd-server -n argocd --timeout=120s
    kubectl apply -f "$SCRIPT_DIR/argocd/argocd-server-nodeport.yaml"
    ```
    Apply the insecure ConfigMap, restart argocd-server to pick it up, wait for it to be ready, then apply the NodePort service.

    **Steps 10-11: KEEP as-is** (build app images, load into kind). Renumber to Step 10 and Step 11.

    **NEW Step 12: Apply ArgoCD root application**
    ```bash
    info "Applying ArgoCD root app-of-apps..."
    kubectl apply -f "$SCRIPT_DIR/argocd/root-app.yaml"
    ```
    This triggers ArgoCD to discover and sync all child Applications, which in turn deploy all infrastructure and application manifests from Git.

    **NEW Step 13: Wait for ArgoCD to sync all applications**
    ```bash
    info "Waiting for ArgoCD to sync all applications..."
    # Wait for all child apps to be created by the root app
    sleep 10

    # Wait for infrastructure pods
    info "Waiting for PostgreSQL..."
    kubectl wait --for=condition=ready pod -l app=postgres -n "$NAMESPACE" --timeout=120s

    info "Waiting for RabbitMQ..."
    kubectl wait --for=condition=ready pod -l app=rabbitmq -n "$NAMESPACE" --timeout=120s

    info "Waiting for Keycloak (may take up to 3 minutes for realm import)..."
    kubectl wait --for=condition=ready pod -l app=keycloak -n "$NAMESPACE" --timeout=180s

    # Wait for application pods
    info "Waiting for ApiService (may take up to 3 minutes for first-boot migrations)..."
    kubectl wait --for=condition=ready pod -l app=apiservice -n "$NAMESPACE" --timeout=180s

    info "Waiting for Gateway..."
    kubectl wait --for=condition=ready pod -l app=gateway -n "$NAMESPACE" --timeout=120s

    info "Waiting for Web..."
    kubectl wait --for=condition=ready pod -l app=web -n "$NAMESPACE" --timeout=120s
    ```
    Keep pod wait commands because the user needs to know when everything is ready. ArgoCD sync triggers deployment, but we still wait for pods to be Running.

    **Step 14: Print access info -- UPDATE** to include ArgoCD:
    Add these lines to the access info section:
    ```bash
    ARGOCD_PASSWORD=$(kubectl -n argocd get secret argocd-initial-admin-secret \
      -o jsonpath='{.data.password}' | base64 --decode)

    # In the echo block, add:
    echo "  ArgoCD UI:           http://localhost:38443 (user: admin, pass: ${ARGOCD_PASSWORD})"
    ```
    Also add to Useful commands:
    ```bash
    echo "  kubectl get applications -n argocd"
    ```
