  <files>src/Directory.Build.props</files>
  <action>
Add a new PropertyGroup to `src/Directory.Build.props` with shared container publishing configuration. Add AFTER the existing PropertyGroup (do not modify existing properties).

New PropertyGroup contents:
```xml
<PropertyGroup>
  <ContainerRegistry>ghcr.io</ContainerRegistry>
  <ContainerBaseImage>mcr.microsoft.com/dotnet/aspnet:10.0-noble-chiseled-extra</ContainerBaseImage>
</PropertyGroup>
```

**Important decisions:**
- Use `noble-chiseled-extra` (NOT plain `noble-chiseled`) because `InvariantGlobalization` is not set to true. The `-extra` variant includes ICU libraries needed for globalization, which EF Core, MassTransit, and Keycloak JWT validation may require.
- Do NOT add `ContainerRuntimeIdentifiers` here — multi-arch is handled in CI via per-arch builds + manifest merge (research found containerd image store dependency for multi-arch dotnet publish).
- Do NOT set `ContainerPort` — .NET 8+ defaults to port 8080 for non-root containers.
- Chiseled images enforce non-root by default (APP_UID=1654), no additional user configuration needed.
