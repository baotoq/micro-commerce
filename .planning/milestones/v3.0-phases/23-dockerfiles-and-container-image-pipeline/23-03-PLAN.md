  <action>
Create `.github/workflows/container-images.yml` with the complete workflow. The workflow has three parallel jobs.

**Workflow structure:**

```yaml
name: Container Images

on:
  push:
    branches: [master]
    tags: ['v*.*.*']

jobs:
  build-web:
    name: Build Web
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Log in to ghcr.io
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Generate image metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ghcr.io/baotoq/micro-commerce/web
          tags: |
            type=sha,prefix=sha-
            type=raw,value=latest,enable=${{ github.ref == format('refs/heads/{0}', 'master') }}
            type=semver,pattern={{version}}

      - name: Set up QEMU
        uses: docker/setup-qemu-action@v3

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Build and push
        uses: docker/build-push-action@v6
        with:
          context: src/MicroCommerce.Web
          platforms: linux/amd64,linux/arm64
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha,scope=web
          cache-to: type=gha,scope=web,mode=max

  build-apiservice:
    name: Build ApiService
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup .NET
        uses: actions/setup-dotnet@v4
        with:
          dotnet-version: 10.0.x

      - name: Log in to ghcr.io
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Generate image metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ghcr.io/baotoq/micro-commerce/apiservice
          tags: |
            type=sha,prefix=sha-
            type=raw,value=latest,enable=${{ github.ref == format('refs/heads/{0}', 'master') }}
            type=semver,pattern={{version}}

      - name: Publish amd64 image
        run: |
          dotnet publish src/MicroCommerce.ApiService /t:PublishContainer \
            --os linux --arch x64 \
            -c Release \
            -p ContainerImageTags='"${{ github.sha }}-amd64"'

      - name: Publish arm64 image
        run: |
          dotnet publish src/MicroCommerce.ApiService /t:PublishContainer \
            --os linux --arch arm64 \
            -c Release \
            -p ContainerImageTags='"${{ github.sha }}-arm64"'

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Create and push multi-arch manifest
        run: |
          TAGS="${{ steps.meta.outputs.tags }}"
          # Build imagetools create command with all tags
          TAG_ARGS=""
          while IFS= read -r tag; do
            [ -n "$tag" ] && TAG_ARGS="$TAG_ARGS -t $tag"
          done <<< "$TAGS"

          docker buildx imagetools create $TAG_ARGS \
            ghcr.io/baotoq/micro-commerce/apiservice:${{ github.sha }}-amd64 \
            ghcr.io/baotoq/micro-commerce/apiservice:${{ github.sha }}-arm64

  build-gateway:
    name: Build Gateway
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup .NET
        uses: actions/setup-dotnet@v4
        with:
          dotnet-version: 10.0.x

      - name: Log in to ghcr.io
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Generate image metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ghcr.io/baotoq/micro-commerce/gateway
          tags: |
            type=sha,prefix=sha-
            type=raw,value=latest,enable=${{ github.ref == format('refs/heads/{0}', 'master') }}
            type=semver,pattern={{version}}

      - name: Publish amd64 image
        run: |
          dotnet publish src/MicroCommerce.Gateway /t:PublishContainer \
            --os linux --arch x64 \
            -c Release \
            -p ContainerImageTags='"${{ github.sha }}-amd64"'

      - name: Publish arm64 image
        run: |
          dotnet publish src/MicroCommerce.Gateway /t:PublishContainer \
            --os linux --arch arm64 \
            -c Release \
            -p ContainerImageTags='"${{ github.sha }}-arm64"'

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Create and push multi-arch manifest
        run: |
          TAGS="${{ steps.meta.outputs.tags }}"
          TAG_ARGS=""
          while IFS= read -r tag; do
            [ -n "$tag" ] && TAG_ARGS="$TAG_ARGS -t $tag"
          done <<< "$TAGS"

          docker buildx imagetools create $TAG_ARGS \
            ghcr.io/baotoq/micro-commerce/gateway:${{ github.sha }}-amd64 \
            ghcr.io/baotoq/micro-commerce/gateway:${{ github.sha }}-arm64
```

**Key design decisions:**
- **New file, not extending release.yml** — release.yml uses 1Password and handles NuGet. Clean separation per research recommendation.
- **GITHUB_TOKEN for ghcr.io auth** — auto-rotated, scoped to repo, no external secret management needed. `permissions: packages: write` grants push access.
- **docker/metadata-action for tags** — declaratively generates SHA, latest, and semver tags. Handles edge cases (PR tags, branch refs) automatically.
- **Three parallel jobs** — all three build concurrently (no dependencies between them in CI). Per locked decision: parallel matrix strategy.
- **.NET multi-arch via per-arch publish + imagetools merge** — avoids containerd image store dependency on GitHub runners (dotnet/sdk#52634). Two `dotnet publish` calls (x64, arm64) produce single-arch images, then `docker buildx imagetools create` merges them into a multi-arch OCI manifest.
- **Next.js multi-arch via QEMU + buildx** — standard Docker approach, fully supported on ubuntu-latest.
- **GHA cache for Next.js** — `cache-from/to: type=gha` uses GitHub Actions cache backend for Docker layer caching, scoped per image.
- **.NET image tags use full SHA for arch-specific** — `${{ github.sha }}-amd64` and `-arm64` are intermediate tags only used for manifest creation. The final user-facing tags come from metadata-action.
- **dotnet-version: 10.0.x** — matches the project's net10.0 target framework.

**Important:** The `ContainerImageTags` value MUST be wrapped in single quotes containing double quotes (`'"value"'`) because bash interprets semicolons. For single tags without semicolons this is technically optional but kept for consistency.
