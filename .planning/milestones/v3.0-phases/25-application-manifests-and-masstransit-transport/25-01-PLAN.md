2. In `Program.cs`, implement conditional transport switching. Read `MASSTRANSIT_TRANSPORT` from configuration. Default to `AzureServiceBus` when not set (preserves existing Aspire workflow).

   **Before the MassTransit registration block**, read the transport:
   ```csharp
   string massTransitTransport = builder.Configuration["MASSTRANSIT_TRANSPORT"] ?? "AzureServiceBus";
   bool useRabbitMq = massTransitTransport.Equals("RabbitMQ", StringComparison.OrdinalIgnoreCase);
   ```

   **Make Azure SDK registrations conditional** -- move `AddAzureBlobServiceClient("blobs")` and `AddAzureServiceBusClient("messaging")` into a conditional block:
   ```csharp
   if (!useRabbitMq)
   {
       builder.AddAzureBlobServiceClient("blobs");
       builder.AddAzureServiceBusClient("messaging");
   }
   ```

   **In the MassTransit `AddMassTransit` block**, replace the hardcoded `x.UsingAzureServiceBus(...)` at the bottom with a conditional:
   ```csharp
   if (useRabbitMq)
   {
       x.UsingRabbitMq((context, cfg) =>
       {
           cfg.Host(builder.Configuration["RabbitMQ:Host"] ?? "localhost", "/", h =>
           {
               h.Username(builder.Configuration["RabbitMQ:Username"] ?? "guest");
               h.Password(builder.Configuration["RabbitMQ:Password"] ?? "guest");
           });
           cfg.ConfigureEndpoints(context);
       });
   }
   else
   {
       x.UsingAzureServiceBus((context, cfg) =>
       {
           cfg.Host(builder.Configuration.GetConnectionString("messaging"));
           cfg.ConfigureEndpoints(context);
       });
   }
   ```
   Keep ALL consumer, saga, outbox, and endpoint callback registrations UNCHANGED -- they are transport-agnostic.

   **Make DLQ service registration conditional:**
   Replace the existing `builder.Services.AddScoped<IDeadLetterQueueService, DeadLetterQueueService>();` with:
   ```csharp
   if (useRabbitMq)
   {
       builder.Services.AddScoped<IDeadLetterQueueService, NoOpDeadLetterQueueService>();
   }
   else
   {
       builder.Services.AddScoped<IDeadLetterQueueService, DeadLetterQueueService>();
   }
   ```

   **Also make Keycloak RequireHttpsMetadata unconditionally false** -- change:
   ```csharp
   if (builder.Environment.IsDevelopment())
   {
       options.RequireHttpsMetadata = false;
   }
   ```
   to just:
   ```csharp
   options.RequireHttpsMetadata = false;
   ```
   Reason: K8s cluster runs Keycloak over HTTP internally. This is a dev-only deployment (kind cluster), not production.

3. In `DeadLetterQueueService.cs`, add `NoOpDeadLetterQueueService` class AFTER the existing `DeadLetterQueueService` class (same file, same namespace):
   ```csharp
   /// <summary>
   /// No-op DLQ service for non-Azure Service Bus environments (e.g., RabbitMQ in K8s).
   /// Returns empty results for all operations.
   /// </summary>
   public sealed class NoOpDeadLetterQueueService : IDeadLetterQueueService
   {
       public Task<IReadOnlyList<DeadLetterMessageDto>> PeekDeadLettersAsync(
           string queueName, int maxMessages = 20, CancellationToken ct = default)
           => Task.FromResult<IReadOnlyList<DeadLetterMessageDto>>([]);

       public Task RetryDeadLetterAsync(
           string queueName, long sequenceNumber, CancellationToken ct = default)
           => Task.CompletedTask;

       public Task<int> PurgeDeadLettersAsync(
           string queueName, CancellationToken ct = default)
           => Task.FromResult(0);

       public Task<IReadOnlyList<string>> GetQueueNamesAsync(CancellationToken ct = default)
           => Task.FromResult<IReadOnlyList<string>>([]);
   }
   ```

IMPORTANT: Do NOT change any consumer, saga, or outbox registrations. The transport switch is transparent to business logic.
