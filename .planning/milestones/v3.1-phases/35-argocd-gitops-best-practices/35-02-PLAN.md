  <action>
1. Create placeholder SealedSecret YAML files that will be overwritten by bootstrap.sh at cluster setup time. These placeholders make the repo self-contained and let ArgoCD see the SealedSecret resources declaratively.

   For each service, create `infra/k8s/base/{service}/sealed-secret.yaml` with a placeholder SealedSecret. Use this template pattern:

   **postgres/sealed-secret.yaml:**
   ```yaml
   # This file is overwritten by bootstrap.sh with cluster-specific sealed values.
   # The placeholder structure lets ArgoCD track the resource declaratively.
   apiVersion: bitnami.com/v1alpha1
   kind: SealedSecret
   metadata:
     name: postgres-credentials
     annotations:
       sealedsecrets.bitnami.com/managed: "true"
   spec:
     encryptedData:
       username: AgAAAA==
       password: AgAAAA==
     template:
       metadata:
         name: postgres-credentials
   ```

   **rabbitmq/sealed-secret.yaml:**
   Same structure, name: `rabbitmq-credentials`, keys: `username`, `password`

   **keycloak/sealed-secret.yaml:**
   Same structure, name: `keycloak-credentials`, keys: `admin-username`, `admin-password`

   **web/sealed-secret.yaml:**
   Same structure, name: `web-secrets`, keys: `auth-secret`, `keycloak-client-secret`

   IMPORTANT: The `AgAAAA==` is a dummy encrypted value. The actual encrypted values are generated per-cluster by `kubeseal` in bootstrap.sh. These placeholders exist solely so the file is in Git and kustomize can reference it.

2. Add `sealed-secret.yaml` to the `resources` list in each base kustomization.yaml:

   For `infra/k8s/base/postgres/kustomization.yaml`, add `- sealed-secret.yaml` to resources (after serviceaccount.yaml).
   Same for rabbitmq, keycloak, and web base kustomization.yaml files.
