  <action>
1. Create `infra/k8s/argocd/project.yaml` — a dedicated ArgoCD AppProject:
   ```yaml
   apiVersion: argoproj.io/v1alpha1
   kind: AppProject
   metadata:
     name: micro-commerce
     namespace: argocd
   spec:
     description: MicroCommerce application project
     sourceRepos:
     - https://github.com/baotoq/micro-commerce.git
     destinations:
     - namespace: micro-commerce
       server: https://kubernetes.default.svc
     - namespace: argocd
       server: https://kubernetes.default.svc
     clusterResourceWhitelist:
     - group: ''
       kind: Namespace
   ```
   The destinations allow micro-commerce namespace (workloads) and argocd namespace (for the root app's child Application resources). clusterResourceWhitelist allows Namespace only (needed by CreateNamespace=true sync option).

2. Update `infra/k8s/argocd/root-app.yaml`:
   - Change `project: default` to `project: micro-commerce`
   - Add retry strategy matching child apps:
     ```yaml
     retry:
       limit: 5
       backoff:
         duration: 5s
         factor: 2
         maxDuration: 3m
     ```
   - Add sync-wave annotation `"0"` (deploys first to create child apps)

3. Add sync-wave annotations to ALL 8 child Application manifests in `infra/k8s/argocd/apps/`. Use this ordering:
   - Wave 1: postgres (database must be first)
   - Wave 2: rabbitmq, keycloak (depend on nothing but should wait for DB)
   - Wave 3: apiservice (depends on postgres, rabbitmq, keycloak)
   - Wave 4: gateway (depends on apiservice)
   - Wave 5: web (depends on gateway, keycloak)
   - Wave 6: otel-collector, aspire-dashboard (observability, last)

   Add to each Application's `metadata.annotations`:
   ```yaml
   annotations:
     argocd.argoproj.io/sync-wave: "N"
   ```

4. Change `project: default` to `project: micro-commerce` in ALL 8 child Application manifests.

5. For `postgres.yaml` only, add `ignoreDifferences` and `RespectIgnoreDifferences`:
   ```yaml
   spec:
     ignoreDifferences:
     - group: apps
       kind: StatefulSet
       jsonPointers:
       - /spec/volumeClaimTemplates/0/apiVersion
       - /spec/volumeClaimTemplates/0/kind
     syncPolicy:
       syncOptions:
       - CreateNamespace=true
       - RespectIgnoreDifferences=true
   ```
   This prevents ArgoCD from reporting OutOfSync due to Kubernetes mutating volumeClaimTemplates metadata fields.

6. Also add `RespectIgnoreDifferences` and similar `ignoreDifferences` to `rabbitmq.yaml` since it is also a StatefulSet with volumeClaimTemplates (added in Phase 34).
