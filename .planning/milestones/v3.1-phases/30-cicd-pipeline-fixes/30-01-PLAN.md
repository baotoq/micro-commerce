<task type="auto">
  <name>Task 3: Fix Dockerfile — use ARG instead of ENV for build-time secrets</name>
  <files>src/MicroCommerce.Web/Dockerfile</files>
  <action>
Update `src/MicroCommerce.Web/Dockerfile` in the builder stage:

Change:
```dockerfile
ENV AUTH_SECRET=placeholder-for-docker-build
ENV AUTH_TRUST_HOST=true
```

To:
```dockerfile
ARG AUTH_SECRET=placeholder-for-docker-build
ARG AUTH_TRUST_HOST=true
```

This ensures the placeholder values are only available at build time and do NOT leak into the final runtime image layers. The real secrets are injected at runtime via Kubernetes Secrets/env vars.

Do NOT add these ARGs to the runner stage — they must not be present at runtime.
  </action>
  <verify>
    <automated>grep -q "^ARG AUTH_SECRET" src/MicroCommerce.Web/Dockerfile && grep -q "^ARG AUTH_TRUST_HOST" src/MicroCommerce.Web/Dockerfile && ! grep -q "^ENV AUTH_SECRET" src/MicroCommerce.Web/Dockerfile && ! grep -q "^ENV AUTH_TRUST_HOST" src/MicroCommerce.Web/Dockerfile && echo "PASS" || echo "FAIL"</automated>
  </verify>
  <done>Dockerfile uses ARG for build-time placeholder secrets; no ENV with secrets leaks to runtime image</done>
