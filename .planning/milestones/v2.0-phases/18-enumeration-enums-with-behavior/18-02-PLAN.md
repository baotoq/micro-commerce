**Step 1: Migrate Order entity methods to TransitionTo()**

Replace the ad-hoc if/throw guards in each Order method with `Status.TransitionTo(target)` followed by `Status = target`. The SmartEnum TransitionTo() already throws InvalidOperationException with a diagnostic message on invalid transitions, so the existing exception behavior is preserved.

Specific method changes:

- `MarkAsPaid()`: Replace `if (Status is not (OrderStatus.Submitted or OrderStatus.StockReserved)) throw ...` with `Status.TransitionTo(OrderStatus.Paid); Status = OrderStatus.Paid;`. Keep `PaidAt = DateTimeOffset.UtcNow;` and the domain event.

- `MarkAsFailed(string reason)`: Replace the complex if check `if (Status is OrderStatus.Confirmed or OrderStatus.Shipped or OrderStatus.Delivered or OrderStatus.Failed) throw ...` with `Status.TransitionTo(OrderStatus.Failed); Status = OrderStatus.Failed;`. Keep the argument guard, FailureReason assignment, and domain event.

- `Confirm()`: Replace `if (Status != OrderStatus.Paid) throw ...` with `Status.TransitionTo(OrderStatus.Confirmed); Status = OrderStatus.Confirmed;`.

- `Ship()`: Replace `if (Status != OrderStatus.Confirmed) throw ...` with `Status.TransitionTo(OrderStatus.Shipped); Status = OrderStatus.Shipped;`.

- `Deliver()`: Replace `if (Status != OrderStatus.Shipped) throw ...` with `Status.TransitionTo(OrderStatus.Delivered); Status = OrderStatus.Delivered;`.

- `MarkStockReserved()`: Replace `if (Status != OrderStatus.Submitted) throw ...` with `Status.TransitionTo(OrderStatus.StockReserved); Status = OrderStatus.StockReserved;`.

Note: `Order.Create()` factory assigns `Status = OrderStatus.Submitted` directly (initial state, no transition) — leave this unchanged.

The `MarkAsFailed` transition check is subtly different with SmartEnum. The current code blocks (Confirmed, Shipped, Delivered, Failed) which means it allows (Submitted, StockReserved, Paid, Cancelled). But per CONTEXT.md, Failed is only reachable from Submitted and StockReserved. The SmartEnum CanTransitionTo rules will enforce the stricter CONTEXT.md rules. The Paid state will NOT be able to transition to Failed under the new rules. Review the existing consumers to verify this is safe — the saga's `OrderFailed` event only fires from Submitted/StockReserved states, so this tightening is correct.

**Step 1: Migrate Product entity methods to TransitionTo()**

Current Product methods have idempotent no-op guards (e.g., `if (Status == ProductStatus.Published) return;`). Per CONTEXT.md, TransitionTo() throws on invalid transitions. Since Published→Published is NOT a valid transition in the defined rules, the behavior changes from silent no-op to throwing.

However, looking at the CONTEXT.md decisions and the Open Question #2 from RESEARCH.md, the entity methods should guard before calling TransitionTo() to preserve reasonable behavior:

- `Publish()`: Change to:
  ```csharp
  if (Status == ProductStatus.Published) return; // idempotent
  Status.TransitionTo(ProductStatus.Published);
  Status = ProductStatus.Published;
  UpdatedAt = DateTimeOffset.UtcNow;
  AddDomainEvent(new ProductStatusChangedDomainEvent(Id, Status));
  ```
  The idempotent guard stays because callers may invoke Publish() without checking current state. The TransitionTo() guard catches actually invalid transitions (e.g., Archived→Published). This gives us: Draft→Published (valid, transitions), Published→Published (no-op), Archived→Published (throws).

- `Unpublish()`: Same pattern — keep `if (Status == ProductStatus.Draft) return;` idempotent guard, then `Status.TransitionTo(ProductStatus.Draft); Status = ProductStatus.Draft;`. This handles: Published→Draft (valid, transitions), Draft→Draft (no-op), Archived→Draft (throws per CONTEXT.md terminal rule).

- `Archive()`: Change to:
  ```csharp
  if (Status == ProductStatus.Archived) return; // idempotent
  Status.TransitionTo(ProductStatus.Archived);
  Status = ProductStatus.Archived;
  UpdatedAt = DateTimeOffset.UtcNow;
  AddDomainEvent(new ProductArchivedDomainEvent(Id));
  ```
  Draft→Archived and Published→Archived are both valid per CONTEXT.md. Archived→Archived is no-op.

**Step 5: Update ChangeProductStatusCommandHandler to support all transitions**

The current handler only supports "Published" and "Unpublish" (defaults to unpublish). With SmartEnum, update to support "Draft", "Published", and "Archived":
```csharp
switch (request.Status.ToLowerInvariant())
{
    case "published":
        product.Publish();
        break;
    case "draft":
        product.Unpublish();
        break;
    case "archived":
        product.Archive();
        break;
    default:
        throw new InvalidOperationException($"Unknown status: {request.Status}");
}
```

**Step 6: Update ChangeProductStatusCommandValidator**

The current validator only allows "Draft" or "Published". Update to also allow "Archived":

```csharp
private static bool BeAValidStatus(string status)
{
    return status.Equals("Draft", StringComparison.OrdinalIgnoreCase) ||
           status.Equals("Published", StringComparison.OrdinalIgnoreCase) ||
           status.Equals("Archived", StringComparison.OrdinalIgnoreCase);
}
```
Update the error message to: `"Status must be 'Draft', 'Published', or 'Archived'."`
