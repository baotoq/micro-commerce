  <action>
    Refactor `GetProductsQueryHandler.Handle()` to replace inline Where chains with specification composition:

    1. Add usings:
       - `using Ardalis.Specification;`
       - `using Ardalis.Specification.EntityFrameworkCore;`
       - `using MicroCommerce.ApiService.Features.Catalog.Application.Specifications;`

    2. Replace the filtering section (lines building `query` with inline Where calls) with spec composition:
       ```
       Specification<Product> spec = new ProductsBaseSpec();

       if (request.CategoryId.HasValue)
           spec = spec.And(new ProductsByCategorySpec(CategoryId.From(request.CategoryId.Value)));

       if (!string.IsNullOrWhiteSpace(request.Status) &&
           ProductStatus.TryFromName(request.Status, ignoreCase: true, out ProductStatus? status))
           spec = spec.And(new ProductByStatusSpec(status!));

       if (!string.IsNullOrWhiteSpace(request.Search))
           spec = spec.And(new ProductSearchSpec(request.Search));
       ```

    3. Replace the count query:
       ```
       int totalCount = await _context.Products
           .AsNoTracking()
           .WithSpecification(spec)
           .CountAsync(cancellationToken);
       ```

    4. Keep sorting logic in the handler (sorting is request-specific with SortBy/SortDirection params — not a specification concern). Apply spec first, then sort:
       ```
       IQueryable<Product> sortedQuery = _context.Products
           .AsNoTracking()
           .WithSpecification(spec);
       ```
       Then apply the existing switch expression for sorting on `sortedQuery`.

    5. Keep pagination (Skip/Take) in the handler — NOT in specs (per research pitfall #1: pagination in specs breaks count queries).

    6. Keep the .Join() for categories and .Select() projection in the handler unchanged — specs handle filtering only, not projections (per research anti-pattern).

    7. Use explicit types per C# conventions: `Specification<Product> spec`, `int totalCount`, `IQueryable<Product> sortedQuery`.

    IMPORTANT: The handler must maintain identical API contract — same ProductListDto response, same pagination, same sorting options.
