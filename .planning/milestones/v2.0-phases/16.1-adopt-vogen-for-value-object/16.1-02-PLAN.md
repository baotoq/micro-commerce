**After generating each migration, inspect the generated `Up()` method:**
- If `Up()` and `Down()` are **empty**: The converter change is metadata-only and doesn't affect the database schema. This is the expected outcome. Keep the migration — it updates the model snapshot to reflect Vogen converters.
- If `Up()` has **column type changes**: Something is wrong. Vogen IDs should still serialize as `uuid`. Investigate before proceeding.
- If migration generation **fails**: Check the error. Common issues:
  - Design-time DbContext creation failure: May need to check Program.cs registration
  - Missing Vogen source generation: Ensure `dotnet build` passes first (Plan 01 must be complete)

**If any migration is completely unnecessary** (tool reports "no changes detected" and does not create files), that DbContext needs no migration. This can happen if the snapshot doesn't track converter implementation details.

**After all 8 migrations are generated, run a final build:**
```bash
dotnet build src/MicroCommerce.ApiService
```

This ensures the migration files compile correctly with the updated model snapshots.
