<objective>
Pilot the Result pattern on two command handlers: UpdateOrderStatusCommandHandler (invalid status transitions) and AdjustStockCommandHandler (negative stock). Change both commands to return `Result` instead of void/Unit, wrap domain method `InvalidOperationException` calls in try-catch to return `Result.Fail`, and update endpoints to use `ToHttpResult()`.

Purpose: Validates the Result infrastructure from Plan 01 works end-to-end with real handlers. Demonstrates the pattern for future adoption (ADOPT-05). Both handlers keep throwing `NotFoundException` and `ConflictException` for infrastructure failures (the ADR boundary).

Output: Two command handlers returning `Result`, two endpoints using `ToHttpResult()`, proving the full pipeline works.
</objective>
