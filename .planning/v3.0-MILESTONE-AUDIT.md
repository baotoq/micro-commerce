## Tech Debt by Phase

### Phase 23: Dockerfiles & CI
- No NuGet restore cache in CI (each build does fresh `dotnet restore`)
- No `--platform=$BUILDPLATFORM` on Dockerfile FROM (slower QEMU builds on ARM64)
- `AUTH_SECRET=placeholder-for-docker-build` in Dockerfile (intentional — prevents next-auth v5 build failure)

### Phase 25: Application Manifests
- ConfigMap not created — env vars inlined in Deployments (documented design decision)
- Resource limits set in base, not patched in dev overlay (documented design decision)
- Pre-existing SixLabors.ImageSharp vulnerability warnings (NU1903/NU1902)
- `services__gateway__https__0` env var name holds `http://` value — functionally correct but naming confusing

### Phase 25.1: UI Refresh
- Hero image placeholder: gray `bg-muted` div instead of actual image
- `AUTH_URL=http://localhost:38800` works for local kind dev only — needs adjustment for remote clusters
- `next.config.ts` image `remotePatterns` reference Azurite — product images from Aspire dev won't resolve in K8s (deferred to future STOR-01)

**Total: 10 items across 3 phases — all non-blocking**
