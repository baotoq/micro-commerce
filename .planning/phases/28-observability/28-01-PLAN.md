  <action>
Create directory `infra/k8s/base/otel-collector/` with four files:

**kustomization.yaml:**
Standard kustomization referencing configmap.yaml, deployment.yaml, service.yaml.

**configmap.yaml:**
ConfigMap named `otel-collector-config` with key `otel-collector-config.yaml` containing the full OTEL Collector pipeline config:
- `receivers.otlp.protocols.grpc.endpoint: 0.0.0.0:4317` and `http.endpoint: 0.0.0.0:4318`
- `processors.memory_limiter`: check_interval 5s, limit_percentage 80, spike_limit_percentage 25
- `processors.batch`: timeout 5s, send_batch_size 1024
- `exporters.otlp/aspire`: endpoint `aspire-dashboard:18889`, tls.insecure: true (CRITICAL -- Dashboard runs plain HTTP internally; without insecure:true the Collector gets TLS handshake errors)
- `extensions.health_check`: endpoint 0.0.0.0:13133
- `service.extensions: [health_check]`
- `service.pipelines`: traces, metrics, and logs -- each with receivers [otlp], processors [memory_limiter, batch] (memory_limiter MUST be first), exporters [otlp/aspire]

**deployment.yaml:**
Deployment named `otel-collector`, labels `app: otel-collector`, 1 replica:
- Container image: `otel/opentelemetry-collector:0.146.0`
- Args: `["--config=/etc/otelcol/otel-collector-config.yaml"]`
- Ports: 4317 (otlp-grpc), 4318 (otlp-http), 13133 (health)
- VolumeMount: name `config`, mountPath `/etc/otelcol`
- Volume: configMap name `otel-collector-config`
- livenessProbe: httpGet path `/` port 13133, periodSeconds 15
- readinessProbe: httpGet path `/` port 13133, periodSeconds 10
- Resources: requests 64Mi/50m, limits 256Mi/200m

**service.yaml:**
ClusterIP Service named `otel-collector`, selector `app: otel-collector`:
- Port 4317 targetPort 4317 name otlp-grpc
- Port 4318 targetPort 4318 name otlp-http

No NodePort -- OTEL Collector is only accessed by ApiService/Gateway within the cluster.

Create directory `infra/k8s/base/aspire-dashboard/` with three files:

**kustomization.yaml:**
Standard kustomization referencing deployment.yaml and service.yaml.

**deployment.yaml:**
Deployment named `aspire-dashboard`, labels `app: aspire-dashboard`, 1 replica:
- Container image: `mcr.microsoft.com/dotnet/aspire-dashboard:9.3`
- Ports: 18888 (ui), 18889 (otlp-grpc)
- Env vars:
  - `DOTNET_DASHBOARD_UNSECURED_ALLOW_ANONYMOUS`: "true" (NOTE: use DOTNET_ prefix, not deprecated ASPIRE_ prefix)
  - `DASHBOARD__OTLP__AUTHMODE`: "Unsecured"
- livenessProbe: httpGet path `/` port 18888, initialDelaySeconds 5, periodSeconds 15
- readinessProbe: httpGet path `/` port 18888, periodSeconds 10
- Resources: requests 128Mi/100m, limits 512Mi/250m

**service.yaml:**
Two Services in one file (separated by `---`):

1. ClusterIP Service named `aspire-dashboard`, selector `app: aspire-dashboard`:
   - Port 18888 targetPort 18888 name ui
   - Port 18889 targetPort 18889 name otlp-grpc

2. NodePort Service named `aspire-dashboard-nodeport`, selector `app: aspire-dashboard`:
   - Port 18888 targetPort 18888 nodePort 30888 name ui

The ClusterIP handles internal OTLP traffic from OTEL Collector. The NodePort enables browser access from the host at localhost:38888 (via kind port mapping).

**Update infra/k8s/base/kustomization.yaml:**
Add `- otel-collector/` and `- aspire-dashboard/` to the resources list, after the existing `- web/` entry. This registers both new services for Kustomize discovery.
