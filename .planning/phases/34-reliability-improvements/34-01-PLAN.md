  <action>
Modify the existing bootstrap.sh to add three safety mechanisms, all inserted BEFORE the "Step 1: Create kind cluster" section:

1. **Required CLI tool check** (immediately after the color/info/warn function definitions):
   - Add a `RED` color variable: `RED='\033[0;31m'`
   - Add an `error()` function: `error() { echo -e "${RED}[ERROR]${NC} $*"; }`
   - Create a `check_prerequisites()` function that:
     - Loops over `kind kubectl kubeseal docker` (docker is also needed for image builds)
     - For each, runs `command -v "$cmd" &>/dev/null` and if missing, prints `error "Required tool '$cmd' not found. Please install it before running this script."` and sets a `missing=true` flag
     - After the loop, if `missing=true`, prints a summary message with install hints:
       ```
       error "Install missing tools:"
       error "  kind:      https://kind.sigs.k8s.io/docs/user/quick-start/#installation"
       error "  kubectl:   https://kubernetes.io/docs/tasks/tools/"
       error "  kubeseal:  https://github.com/bitnami-labs/sealed-secrets#kubeseal"
       error "  docker:    https://docs.docker.com/get-docker/"
       ```
     - Then `exit 1`
   - Call `check_prerequisites` right after defining it

2. **Trap handler** (after prerequisites check, before Step 1):
   - Define a `cleanup_on_error()` function that prints:
     ```
     echo ""
     error "Bootstrap failed! To clean up:"
     error "  kind delete cluster --name ${CLUSTER_NAME}"
     error "  kubectl config unset contexts.kind-${CLUSTER_NAME}"
     echo ""
     error "To retry, run this script again."
     ```
   - Set trap: `trap cleanup_on_error ERR`

3. **Context guard** (after the kind cluster creation in Step 1, replacing current Step 2):
   - After cluster creation/skip, get current context: `CURRENT_CONTEXT=$(kubectl config current-context 2>/dev/null || echo "")`
   - Expected context: `EXPECTED_CONTEXT="kind-${CLUSTER_NAME}"`
   - If `CURRENT_CONTEXT != EXPECTED_CONTEXT`, print error and exit:
     ```
     error "kubectl context is '${CURRENT_CONTEXT}' but expected '${EXPECTED_CONTEXT}'"
     error "This script only runs against the local kind cluster."
     error "Switch context: kubectl config use-context ${EXPECTED_CONTEXT}"
     exit 1
     ```
   - Keep the existing `kubectl config use-context` call after the guard passes

Do NOT change any other parts of the script (sealed secrets, ArgoCD, image builds, etc.).
