**2. `infra/k8s/base/rabbitmq/deployment.yaml`:**
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: rabbitmq
  namespace: micro-commerce
spec:
  replicas: 1
  selector:
    matchLabels:
      app: rabbitmq
  template:
    metadata:
      labels:
        app: rabbitmq
    spec:
      containers:
      - name: rabbitmq
        image: rabbitmq:4-management
        ports:
        - containerPort: 5672
          name: amqp
        - containerPort: 15672
          name: management
        env:
        - name: RABBITMQ_DEFAULT_USER
          valueFrom:
            secretKeyRef:
              name: rabbitmq-credentials
              key: username
        - name: RABBITMQ_DEFAULT_PASS
          valueFrom:
            secretKeyRef:
              name: rabbitmq-credentials
              key: password
        readinessProbe:
          exec:
            command: ["rabbitmq-diagnostics", "ping"]
          initialDelaySeconds: 10
          periodSeconds: 10
        livenessProbe:
          exec:
            command: ["rabbitmq-diagnostics", "status"]
          initialDelaySeconds: 30
          periodSeconds: 30
        resources:
          requests:
            memory: 256Mi
            cpu: 250m
          limits:
            memory: 512Mi
            cpu: 500m
```

Key decisions:
- `rabbitmq:4-management` image includes the management plugin (UI at port 15672).
- Credentials from `rabbitmq-credentials` Secret (will be sealed: guest/guest for dev).
- Deployment (not StatefulSet) — single-node dev, no persistence needed for message broker in dev.
- Resource limits: 256Mi-512Mi memory, 250m-500m CPU.

**3. `infra/k8s/base/keycloak/deployment.yaml`:**
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: keycloak
  namespace: micro-commerce
spec:
  replicas: 1
  selector:
    matchLabels:
      app: keycloak
  template:
    metadata:
      labels:
        app: keycloak
    spec:
      containers:
      - name: keycloak
        image: quay.io/keycloak/keycloak:26.0
        args: ["start-dev", "--import-realm"]
        env:
        - name: KC_BOOTSTRAP_ADMIN_USERNAME
          valueFrom:
            secretKeyRef:
              name: keycloak-credentials
              key: admin-username
        - name: KC_BOOTSTRAP_ADMIN_PASSWORD
          valueFrom:
            secretKeyRef:
              name: keycloak-credentials
              key: admin-password
        - name: KC_HEALTH_ENABLED
          value: "true"
        ports:
        - containerPort: 8080
          name: http
        - containerPort: 9000
          name: management
        volumeMounts:
        - name: realm-config
          mountPath: /opt/keycloak/data/import
        startupProbe:
          httpGet:
            path: /health/started
            port: 9000
          initialDelaySeconds: 15
          periodSeconds: 3
          failureThreshold: 20
        livenessProbe:
          httpGet:
            path: /health/live
            port: 9000
          periodSeconds: 15
        readinessProbe:
          httpGet:
            path: /health/ready
            port: 9000
          periodSeconds: 10
        resources:
          requests:
            memory: 512Mi
            cpu: 500m
          limits:
            memory: 1Gi
            cpu: "1"
      volumes:
      - name: realm-config
        configMap:
          name: keycloak-realm
```

Key decisions:
- `start-dev` mode — no TLS, fast startup, appropriate for local kind cluster (per user decision).
- `--import-realm` flag — imports realm JSON from `/opt/keycloak/data/import/` on first boot. Uses IGNORE_EXISTING strategy (idempotent on restarts).
- `KC_BOOTSTRAP_ADMIN_USERNAME` / `KC_BOOTSTRAP_ADMIN_PASSWORD` — Keycloak 26.x env vars (NOT the deprecated `KEYCLOAK_ADMIN`).
- `KC_HEALTH_ENABLED=true` — enables health endpoints on management port 9000.
- **Startup probe:** `initialDelaySeconds: 15`, `periodSeconds: 3`, `failureThreshold: 20` = 15s delay + up to 60s of probing = 75s total window. Per user decision: "15s initial delay, 60s failure threshold timeout". While startup probe runs, liveness/readiness are disabled.
- Health checks on management port 9000 (NOT main port 8080) — Keycloak 26.x serves health on management interface.
- Higher resource limits than RabbitMQ (512Mi-1Gi memory, 500m-1 CPU) — Keycloak is heavier with realm import.
- ConfigMap name `keycloak-realm` — Kustomize will auto-append hash suffix and update the Deployment reference.
