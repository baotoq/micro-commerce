Create a new file `src/MicroCommerce.ApiService.Tests/Integration/Interceptors/InterceptorBehaviorTests.cs`.

This test class verifies that the three interceptors actually fire at runtime by using Cart entities (Cart implements both `AuditableAggregateRoot<CartId>` (which provides IAuditable) and `IConcurrencyToken`).

**Test class structure:**
- `[Collection("Integration Tests")]` and `[Trait("Category", "Integration")]`
- Extends `IntegrationTestBase`
- Constructor receives `ApiWebApplicationFactory factory` and passes to base
- `override InitializeAsync()` calls `await ResetDatabase(typeof(CartDbContext));`

**Required using statements:**
```csharp
using FluentAssertions;
using MicroCommerce.ApiService.Features.Cart.Domain.Entities;
using MicroCommerce.ApiService.Features.Cart.Infrastructure;
using MicroCommerce.ApiService.Tests.Integration.Fixtures;
using Microsoft.EntityFrameworkCore;
using Microsoft.Extensions.DependencyInjection;
```

**Test 1: `AuditInterceptor_OnInsert_SetsCreatedAtAndUpdatedAt`**
- Record `DateTimeOffset before = DateTimeOffset.UtcNow.AddSeconds(-1);`
- Create scope, get `CartDbContext`
- Create `Cart.Create(Guid.NewGuid())`, add to context, `SaveChangesAsync()`
- Assert `cart.CreatedAt.Should().BeAfter(before)`
- Assert `cart.UpdatedAt.Should().BeAfter(before)`
- Assert `cart.CreatedAt.Should().Be(cart.UpdatedAt)` (on insert they're equal)

**Test 2: `AuditInterceptor_OnUpdate_UpdatesUpdatedAtOnly`**
- Create scope, get `CartDbContext`
- Create and save a cart (insert)
- Record `DateTimeOffset originalCreatedAt = cart.CreatedAt;`
- `await Task.Delay(50);` (ensure clock difference for UpdatedAt)
- Modify the cart: `cart.AddItem(Guid.NewGuid(), "Test Product", 9.99m, null, 1);`
- `await db.SaveChangesAsync();`
- Assert `cart.CreatedAt.Should().Be(originalCreatedAt)` (unchanged)
- Assert `cart.UpdatedAt.Should().BeAfter(originalCreatedAt)` (updated)

**Test 3: `ConcurrencyInterceptor_OnInsert_SetsVersionToOne`**
- Create scope, get `CartDbContext`
- Create and save a cart
- Assert `cart.Version.Should().Be(1)`

**Test 4: `ConcurrencyInterceptor_OnUpdate_IncrementsVersion`**
- Create scope, get `CartDbContext`
- Create and save a cart (Version should be 1)
- Modify the cart: `cart.AddItem(Guid.NewGuid(), "Test Product", 9.99m, null, 1);`
- `await db.SaveChangesAsync();`
- Assert `cart.Version.Should().Be(2)`

**Test 5: `ConcurrencyInterceptor_ConcurrentUpdate_ThrowsDbUpdateConcurrencyException`**
- Insert a cart in scope 0, record the `CartId cartId`
- Open scope1 with its own `CartDbContext db1`, read cart via `await db1.Carts.AsTracking().FirstAsync(c => c.Id == cartId);` (tracked, Version=1)
- Open scope2 with its own `CartDbContext db2`, read same cart (tracked, Version=1)
- Modify cart2: `cart2.AddItem(...)`, save via `db2.SaveChangesAsync()` — succeeds, Version goes 1->2
- Modify cart1: `cart1.AddItem(...)`, attempt save via `db1.SaveChangesAsync()` — stale Version=1 vs DB Version=2
- Assert `Func<Task> act = async () => await db1.SaveChangesAsync(); await act.Should().ThrowAsync<DbUpdateConcurrencyException>();`

**IMPORTANT usage notes for Cart entity:**
- `Cart.Create(Guid buyerId)` takes a raw `Guid`, NOT a BuyerId value object
- `cart.AddItem(Guid productId, string productName, decimal unitPrice, string? imageUrl, int quantity)` — productId is raw Guid, all params required
- Use `Guid.NewGuid()` for buyer and product IDs in tests
