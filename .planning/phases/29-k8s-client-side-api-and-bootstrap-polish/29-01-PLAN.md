1. **Delete line 1** (`const API_BASE = process.env.NEXT_PUBLIC_API_URL || 'http://localhost:5200';`)

2. **Add the runtime config resolver** at the top of the file (before any interfaces):

```typescript
// Runtime-resolved API base URL (fetched once from /api/config)
let _configPromise: Promise<string> | null = null;

function getApiBase(): Promise<string> {
  if (!_configPromise) {
    _configPromise =
      typeof window !== "undefined"
        ? fetch("/api/config")
            .then((r) => r.json())
            .then((d) => d.apiBaseUrl as string)
            .catch(() => "http://localhost:5200")
        : Promise.resolve(
            process.env.NEXT_PUBLIC_API_URL || "http://localhost:5200"
          );
  }
  return _configPromise;
}
```

Key design decisions:
- **Promise-based singleton**: Caches the Promise itself (not the resolved value) to prevent race conditions when multiple components mount simultaneously
- **`typeof window !== 'undefined'` guard**: Prevents SSR from trying to fetch its own route handler via HTTP
- **Server-side fallback**: Uses `process.env.NEXT_PUBLIC_API_URL` for backward compatibility in SSR, falls back to localhost for local dev
- **catch fallback**: If `/api/config` fails (e.g., offline dev), falls back to localhost:5200

3. **Mechanically replace every `${API_BASE}` usage** in every function with:
   - Add `const apiBase = await getApiBase();` as the FIRST line of every async function that uses the base URL
   - Replace `${API_BASE}` with `${apiBase}`

There are approximately 35+ functions that use `API_BASE`. Every single one must be updated. The change is purely mechanical -- the function signatures, return types, and behavior remain identical. The hooks in `src/hooks/` already call these functions via TanStack Query's `queryFn` which expects Promise returns, so NO hook changes are needed.
