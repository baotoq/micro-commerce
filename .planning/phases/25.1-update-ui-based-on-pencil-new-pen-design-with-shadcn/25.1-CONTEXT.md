# Phase 25.1: Update UI based on pencil-new.pen design with shadcn - Context

**Gathered:** 2026-03-01
**Status:** Ready for planning

<domain>
## Phase Boundary

Refresh all storefront and admin UI screens to match the pencil-new.pen design using existing shadcn/ui primitives and Tailwind CSS. This is a visual update of existing screens — no new backend features, API endpoints, or routes are added. The design file (`pencil-new.pen`) is the source of truth for layout, colors, and component styling.

</domain>

<decisions>
## Implementation Decisions

### Screen priority & scope
- All 10 screens are in scope: Homepage, Product Catalog, Product Detail, Shopping Cart, Checkout, Order Confirmation, Order History, Auth Login, User Profile, Admin Dashboard
- Admin Dashboard: match pencil design exactly — dark sidebar, table-based product list, badge statuses. Existing admin routes stay, layout gets restyled
- Auth Login: build the custom split-screen login page from pencil design. NextAuth + Keycloak handles auth behind the scenes, but the visual is the custom split layout (blue gradient left, form right)
- Homepage hero: use static curated images. Featured products section below pulls real data from API

### Design fidelity
- Follow the "spirit of the design" — match layout structure, color palette, component style from pencil. Use shadcn defaults for spacing/radius/shadows where close enough. Don't chase pixel-perfect
- Desktop-first with basic responsive: build 1440px layout from pencil, add sensible breakpoints (stack grids to single column, collapse sidebar to sheet/drawer, adjust padding)
- Adopt Inter font via next/font/google
- Standardize on lucide-react icons (shadcn already uses Lucide internally)

### Component strategy
- Custom components (CategoryPill, StarRating, QuantityStepper, StepIndicator, ProductCard) live in `src/components/storefront/` for feature-specific and `src/components/ui/` for reusable primitives
- Install additional shadcn components as needed per screen (e.g., tabs, pagination, breadcrumb). Don't pre-install everything
- Rewrite existing storefront components to match the design — keep same file names and prop interfaces where possible, but JSX/styling gets a fresh start
- Admin sidebar: shared layout component in admin `layout.tsx` as persistent navigation shell

### Color & theming
- Adopt pencil color palette: primary blue #2563EB, foreground #1A1A1A, border #E5E7EB, etc. Map all pencil variables to shadcn/Tailwind CSS custom properties
- Extend theme with semantic status variables: --success, --warning, --error, --info (with background + foreground pairs) for badges and order statuses
- Add sidebar-specific CSS variables (--sidebar-bg #111827, --sidebar-foreground, --sidebar-accent) for admin dashboard dark sidebar
- No dark mode — light mode only. Pencil design only shows light mode

### Claude's Discretion
- Exact responsive breakpoint values and mobile layout adjustments
- Loading/skeleton states per screen
- Error state handling
- Animation/transitions between states
- Exact shadcn component selection per screen
- Which components are "reusable primitives" (ui/) vs "feature components" (storefront/)
- Per-component decision on refactor vs rewrite based on complexity

</decisions>

<specifics>
## Specific Ideas

- The pencil design uses a gradient hero background (blue-to-green, 135deg) on the homepage
- Product Catalog has a left sidebar with category checkboxes, price range, and rating filters with pagination at the bottom
- Product Detail has an image gallery with thumbnails, a tab section (Description / Reviews & Ratings), and an inline review submission
- Checkout uses a step indicator pattern (Shipping -> Payment -> Review) with the order summary sidebar
- Admin Dashboard uses a dark navy sidebar (#111827) with active state highlighting
- Category pills use a pill shape (border-radius 9999) with icon + label, active state uses primary fill
- The design font stack is DM Sans / Inter fallback

</specifics>
