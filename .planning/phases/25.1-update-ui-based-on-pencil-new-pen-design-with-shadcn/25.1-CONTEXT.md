<domain>
## Phase Boundary

Refresh all storefront and admin UI screens to match the split per-page pencil design files using existing shadcn/ui primitives and Tailwind CSS. This is a visual update of existing screens — no new backend features, API endpoints, or routes are added. The `design/*.pen` files are the source of truth for layout, colors, and component styling.

<decisions>
## Implementation Decisions

### Design file strategy
- Source of truth: per-page `.pen` files in `design/` directory (11 files total)
- `design/design-system.pen` for shared component specs (26 reusable components) and color variables
- Each screen references its own `.pen` file (e.g., `design/storefront-homepage.pen` for homepage)
- Old monolithic `pencil-new.pen` is deleted — split files replace it entirely
- Design fidelity: match layout structure, color palette, component style from pencil. Use shadcn defaults for spacing/radius where close enough. Don't chase pixel-perfect

### Component strategy
- Use shadcn/ui for standard components (Button, Badge, Input, Table, Avatar, Tabs, etc.) — theme them to match design
- Create custom components only for things shadcn doesn't cover: CategoryPill, StarRating, QuantityStepper, StepIndicator, ProductCard
- Custom feature components in `src/components/storefront/`, reusable primitives in `src/components/ui/`
- Rewrite existing storefront components to match design — keep same file names and prop interfaces where possible, JSX/styling gets fresh start
- Install additional shadcn components as needed per screen (e.g., tabs, pagination, breadcrumb). Don't pre-install everything

### Font & typography
- Primary font: DM Sans via next/font/google (matches design-system.pen exactly)
- Fallback: Inter, then system sans-serif
- Load weights 400 (normal), 500 (medium), 600 (semibold) only — matches design usage, keeps bundle small
- Single font stack for entire app — no separate display font for headings
- Replace current Geist Sans references with DM Sans

### Color theme migration
- Convert design hex colors to oklch format in globals.css — maintains Tailwind v4 + shadcn best practice
- Primary blue: #2563EB (oklch equivalent), foreground: #1A1A1A, border: #E5E7EB, etc.
- Extend CSS variables with semantic status colors: --success, --success-bg, --success-foreground (and warning/error/info equivalents)
- Add --star (#FBBF24 amber) as a theme variable for star ratings
- Admin sidebar: override --sidebar-bg (#111827), --sidebar-foreground (#F9FAFB), --sidebar-accent (#1F2937) via CSS custom properties
- No dark mode — light mode only. Design only shows light mode
- Remove .dark section from globals.css (or keep inert)

### Screen priority & scope
- All 10 screens + theme/globals setup = 11 implementation units
- One plan per screen for clear progress tracking
- Implementation order (customer journey):
  1. Theme/globals (CSS variables, font, shared component theming)
  2. Homepage (`design/storefront-homepage.pen`)
  3. Product Catalog (`design/storefront-catalog.pen`)
  4. Product Detail (`design/storefront-product-detail.pen`)
  5. Shopping Cart (`design/storefront-cart.pen`)
  6. Checkout (`design/storefront-checkout.pen`)
  7. Order Confirmation (`design/storefront-order-confirmation.pen`)
  8. Order History (`design/storefront-order-history.pen`)
  9. User Profile (`design/storefront-profile.pen`)
  10. Auth Login (`design/auth-login.pen`)
  11. Admin Dashboard (`design/admin-dashboard.pen`)

### Responsive approach
- Desktop-first: build 1440px layout from pencil, add sensible breakpoints
- Stack grids to single column, collapse sidebar to sheet/drawer, adjust padding
- Admin sidebar: shared layout component in admin `layout.tsx` as persistent navigation shell

### Claude's Discretion
- Exact responsive breakpoint values and mobile layout adjustments
- Loading/skeleton states per screen
- Error state handling
- Animation/transitions between states
- Exact shadcn component selection per screen
- Which components are "reusable primitives" (ui/) vs "feature components" (storefront/)
- Per-component decision on refactor vs rewrite based on complexity
- Exact oklch conversion values for each hex color

</decisions>

## Specific Ideas

- The pencil design uses a gradient hero background (blue-to-green, 135deg) on the homepage
- Product Catalog has a left sidebar with category checkboxes, price range, and rating filters with pagination at the bottom
- Product Detail has an image gallery with thumbnails, a tab section (Description / Reviews & Ratings), and an inline review submission
- Checkout uses a step indicator pattern (Shipping -> Payment -> Review) with the order summary sidebar
- Admin Dashboard uses a dark navy sidebar (#111827) with active state highlighting (--sidebar-accent: #1F2937)
- Category pills use pill shape (border-radius 9999) with icon + label, active state uses primary fill
- Design font stack is DM Sans / Inter fallback
- Design system has 26 reusable components: Button (Default, Outline, Ghost, Destructive, Secondary), Badge (Success, Warning, Error, Info, Gray), Input Group, SearchBar, StarRating, ProductCard, Avatar/Text, SidebarNavItem (Active/Inactive), QuantityStepper, CategoryPill (Active/Inactive), Tab (Active/Inactive), TableHeaderRow, TableDataRow, StepIndicator (Active/Inactive)
