  <action>
    Append a new `update-manifests` job to the existing `.github/workflows/container-images.yml` workflow file. Add it after the `build-gateway` job (at the end of the jobs section).

    The new job definition:
    ```yaml
    update-manifests:
      name: Update Kustomize Manifests
      needs: [build-web, build-apiservice, build-gateway]
      if: github.ref == 'refs/heads/master'
      runs-on: ubuntu-latest
      permissions:
        contents: write
      steps:
        - name: Checkout
          uses: actions/checkout@v4

        - name: Setup Kustomize
          uses: imranismail/setup-kustomize@v2

        - name: Update image tags in dev overlay
          run: |
            SHORT_SHA="sha-$(echo '${{ github.sha }}' | cut -c1-7)"
            echo "Updating image tags to: $SHORT_SHA"

            cd infra/k8s/overlays/dev/apiservice
            kustomize edit set image apiservice=ghcr.io/baotoq/micro-commerce/apiservice:$SHORT_SHA

            cd ../gateway
            kustomize edit set image gateway=ghcr.io/baotoq/micro-commerce/gateway:$SHORT_SHA

            cd ../web
            kustomize edit set image web=ghcr.io/baotoq/micro-commerce/web:$SHORT_SHA

        - name: Commit and push
          run: |
            git config user.name "github-actions[bot]"
            git config user.email "github-actions[bot]@users.noreply.github.com"
            git add infra/k8s/overlays/dev/
            git diff --staged --quiet && echo "No changes to commit" && exit 0
            git commit -m "[skip ci] chore: update image tags to sha-$(echo '${{ github.sha }}' | cut -c1-7)"
            git push
    ```

    Key design decisions:
    1. **`needs: [build-web, build-apiservice, build-gateway]`** -- only runs after ALL 3 images are successfully built and pushed. If any build fails, manifests are not updated.
    2. **`if: github.ref == 'refs/heads/master'`** -- prevents manifest updates on tag pushes (`v*.*.*`), which are release artifacts, not deployments.
    3. **`permissions: contents: write`** -- job-level permission, scoped only to this job. The existing build jobs keep their `contents: read` + `packages: write`.
    4. **`imranismail/setup-kustomize@v2`** -- installs the standalone kustomize CLI binary. The `ubuntu-latest` GitHub Actions runner does NOT ship with standalone `kustomize` (it only ships `kubectl`, and `kubectl kustomize` is read-only -- it does not support `edit` subcommands). This step MUST run before any `kustomize edit set image` command.
    5. **`kustomize edit set image <oldName>=<newName>:<newTag>`** -- updates both `newName` (to ghcr.io registry path) and `newTag` (to SHA) in one command. This is the official Kustomize CLI pattern, not fragile sed.
    6. **Short SHA format: `sha-$(echo '${{ github.sha }}' | cut -c1-7)`** -- matches the exact format produced by `docker/metadata-action`'s `type=sha,prefix=sha-` (7-character short SHA with `sha-` prefix).
    7. **`git diff --staged --quiet && echo "No changes" && exit 0`** -- prevents empty commits if the workflow is re-run on the same commit (idempotent).
    8. **`[skip ci]` in commit message** -- belt-and-suspenders protection against infinite loops. GITHUB_TOKEN already prevents re-triggering push-based workflows, but `[skip ci]` is an additional safety net.
    9. **No PAT** -- the default GITHUB_TOKEN is used (no explicit token configuration needed). GitHub Actions does not trigger push-based workflows for commits made by GITHUB_TOKEN.

    DO NOT modify any existing job (build-web, build-apiservice, build-gateway). Only append the new job.
  </action>
