**Create `src/MicroCommerce.Web/Dockerfile`:**

Three-stage Dockerfile: deps (install), builder (build), runner (production).

```dockerfile
FROM node:22-alpine AS base

# Stage 1: Install dependencies
FROM base AS deps
RUN apk add --no-cache libc6-compat
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci --ignore-scripts

# Stage 2: Build application
FROM base AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .

ENV NEXT_TELEMETRY_DISABLED=1

# Placeholder for build-time auth — real secret injected at runtime via K8s Secrets
ENV AUTH_SECRET=placeholder-for-docker-build
ENV AUTH_TRUST_HOST=true

RUN npm run build

# Stage 3: Production runner
FROM base AS runner
WORKDIR /app

ENV NODE_ENV=production
ENV NEXT_TELEMETRY_DISABLED=1
ENV PORT=3000
ENV HOSTNAME=0.0.0.0

RUN addgroup --system --gid 1001 nodejs && \
    adduser --system --uid 1001 nextjs

# Copy static assets and standalone server
COPY --from=builder --chown=nextjs:nodejs /app/public ./public
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static

USER nextjs

EXPOSE 3000

CMD ["node", "server.js"]
```

**Key design decisions:**
- `libc6-compat` on Alpine required for Node.js native module compatibility
- `npm ci --ignore-scripts` for reproducible installs without postinstall side effects
- `AUTH_SECRET=placeholder-for-docker-build` prevents next-auth v5 build failures — the real secret is injected at runtime (Phase 25)
- `AUTH_TRUST_HOST=true` prevents NEXTAUTH_URL requirement at build time
- `.next/static` and `public` MUST be copied explicitly — standalone output excludes them
- `NEXT_TELEMETRY_DISABLED=1` prevents build-time telemetry calls
- Non-root `nextjs` user (UID 1001) matches the official Next.js Docker example
- No HEALTHCHECK — Kubernetes probes handle this (Phase 25, per locked decision)
