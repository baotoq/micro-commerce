<task type="auto">
  <name>Task 2: Add path filtering to container-images.yml</name>
  <files>.github/workflows/container-images.yml</files>
  <action>
Update the `on.push` trigger in container-images.yml to add path filters so container builds only run when source code changes:

```yaml
on:
  push:
    branches: [master]
    tags: ['v*.*.*']
    paths:
      - 'src/**'
      - '.github/workflows/container-images.yml'
      - '**/Dockerfile'
      - '**/Directory.Build.props'
      - '**/Directory.Packages.props'
```

This ensures:
- Pushes to master that only change docs (`*.md`), K8s manifests (`k8s/**`), planning files (`.planning/**`), or other non-source files will NOT trigger container builds.
- Tag pushes (`v*.*.*`) ALWAYS trigger builds regardless of paths (GitHub Actions ignores path filters for tag events, which is the correct behavior).
- Changes to Dockerfiles, build props, workflow file itself, or any source code DO trigger builds.

Do NOT add `paths-ignore` — use inclusive `paths` instead (safer, explicit allowlist).
  </action>
  <verify>
    <automated>grep -A 8 "on:" .github/workflows/container-images.yml | grep -c "paths:"</automated>
  </verify>
  <done>container-images.yml has path filters that skip builds for non-source changes (docs, manifests, planning files). Tag-triggered builds are unaffected.</done>
</task>
