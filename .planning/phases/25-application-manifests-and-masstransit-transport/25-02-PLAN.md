**ApiService Deployment** (`infra/k8s/base/apiservice/deployment.yaml`):
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: apiservice
  labels:
    app: apiservice
spec:
  replicas: 1
  selector:
    matchLabels:
      app: apiservice
  template:
    metadata:
      labels:
        app: apiservice
    spec:
      containers:
      - name: apiservice
        image: apiservice:dev
        ports:
        - containerPort: 8080
        env:
        - name: ASPNETCORE_ENVIRONMENT
          value: Production
        - name: POSTGRES_USERNAME
          valueFrom:
            secretKeyRef:
              name: postgres-credentials
              key: username
        - name: POSTGRES_PASSWORD
          valueFrom:
            secretKeyRef:
              name: postgres-credentials
              key: password
        - name: ConnectionStrings__appdb
          value: "Host=postgres.micro-commerce.svc.cluster.local;Database=appdb;Username=$(POSTGRES_USERNAME);Password=$(POSTGRES_PASSWORD)"
        - name: MASSTRANSIT_TRANSPORT
          value: RabbitMQ
        - name: RabbitMQ__Host
          value: rabbitmq.micro-commerce.svc.cluster.local
        - name: RABBITMQ_USERNAME
          valueFrom:
            secretKeyRef:
              name: rabbitmq-credentials
              key: username
        - name: RABBITMQ_PASSWORD
          valueFrom:
            secretKeyRef:
              name: rabbitmq-credentials
              key: password
        - name: RabbitMQ__Username
          value: "$(RABBITMQ_USERNAME)"
        - name: RabbitMQ__Password
          value: "$(RABBITMQ_PASSWORD)"
        - name: ConnectionStrings__keycloak
          value: "http://keycloak.micro-commerce.svc.cluster.local:8080"
        startupProbe:
          httpGet:
            path: /alive
            port: 8080
          initialDelaySeconds: 10
          periodSeconds: 5
          failureThreshold: 30
        livenessProbe:
          httpGet:
            path: /alive
            port: 8080
          periodSeconds: 15
        readinessProbe:
          httpGet:
            path: /health
            port: 8080
          periodSeconds: 10
        resources:
          requests:
            memory: 256Mi
            cpu: 250m
          limits:
            memory: 512Mi
            cpu: 500m
```

Key design decisions:
- `POSTGRES_USERNAME` and `POSTGRES_PASSWORD` defined BEFORE `ConnectionStrings__appdb` so `$(...)` env var substitution works
- `RABBITMQ_USERNAME` and `RABBITMQ_PASSWORD` from SealedSecret, then mapped to `RabbitMQ__Username` and `RabbitMQ__Password` for MassTransit configuration
- Startup probe: 10s initial delay + 30 * 5s = 160s total window for EF Core migrations on first boot (INFRA-05 deferred from Phase 24)
- `ConnectionStrings__keycloak` provides the Aspire Keycloak integration with the Keycloak base URL for JWT discovery

**Gateway Deployment** (`infra/k8s/base/gateway/deployment.yaml`):
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: gateway
  labels:
    app: gateway
spec:
  replicas: 1
  selector:
    matchLabels:
      app: gateway
  template:
    metadata:
      labels:
        app: gateway
    spec:
      containers:
      - name: gateway
        image: gateway:dev
        ports:
        - containerPort: 8080
        env:
        - name: ASPNETCORE_ENVIRONMENT
          value: Production
        - name: ReverseProxy__Clusters__apiservice__Destinations__default__Address
          value: "http://apiservice:8080"
        - name: ConnectionStrings__keycloak
          value: "http://keycloak.micro-commerce.svc.cluster.local:8080"
        - name: Cors__Origins__0
          value: "http://localhost:38800"
        livenessProbe:
          httpGet:
            path: /alive
            port: 8080
          periodSeconds: 15
        readinessProbe:
          httpGet:
            path: /health
            port: 8080
          periodSeconds: 10
        resources:
          requests:
            memory: 128Mi
            cpu: 100m
          limits:
            memory: 256Mi
            cpu: 250m
```

Key design decisions:
- YARP cluster destination overridden via `ReverseProxy__Clusters__apiservice__Destinations__default__Address` to use K8s short DNS name `apiservice` (same namespace)
- CORS origin set to `http://localhost:38800` (the kind NodePort host mapping for Gateway)
- No startup probe needed -- Gateway starts fast (no DB migrations)
- Lighter resource limits than ApiService (Gateway is a proxy, no DB/message processing)

**Web Deployment** (`infra/k8s/base/web/deployment.yaml`):
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web
  labels:
    app: web
spec:
  replicas: 1
  selector:
    matchLabels:
      app: web
  template:
    metadata:
      labels:
        app: web
    spec:
      containers:
      - name: web
        image: web:dev
        ports:
        - containerPort: 3000
        env:
        - name: NODE_ENV
          value: production
        - name: HOSTNAME
          value: "0.0.0.0"
        - name: PORT
          value: "3000"
        - name: AUTH_SECRET
          value: "k8s-dev-auth-secret-change-in-production"
        - name: AUTH_TRUST_HOST
          value: "true"
        - name: AUTH_URL
          value: "http://localhost:38800"
        - name: KEYCLOAK_ISSUER
          value: "http://keycloak.micro-commerce.svc.cluster.local:8080/realms/micro-commerce"
        - name: KEYCLOAK_CLIENT_ID
          value: "nextjs-app"
        - name: KEYCLOAK_CLIENT_SECRET
          value: "nextjs-app-secret"
        - name: services__gateway__https__0
          value: "http://gateway:8080"
        livenessProbe:
          httpGet:
            path: /
            port: 3000
          initialDelaySeconds: 5
          periodSeconds: 15
        readinessProbe:
          httpGet:
            path: /
            port: 3000
          initialDelaySeconds: 5
          periodSeconds: 10
        resources:
          requests:
            memory: 128Mi
            cpu: 100m
          limits:
            memory: 256Mi
            cpu: 250m
```

Key design decisions:
- `AUTH_URL` set to `http://localhost:38800` (Gateway NodePort host URL) -- NextAuth uses this for OAuth callback URL construction
- `AUTH_TRUST_HOST=true` because there's a reverse proxy (Gateway) in front
- `KEYCLOAK_ISSUER` uses K8s internal DNS for server-side token exchange
- `services__gateway__https__0` tells the Next.js app where to find the Gateway (Aspire-style env var that the frontend `/api/config` route reads)
- Health probes on `/` (Next.js homepage) -- no custom health endpoint needed
- `AUTH_SECRET` and `KEYCLOAK_CLIENT_SECRET` use dev placeholder values. In production, these would be SealedSecrets. For the dev kind cluster, plaintext is acceptable (matching Phase 23 Dockerfile approach with `placeholder-for-docker-build`)
