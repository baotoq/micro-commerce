## Decisions Made
- **CheckoutState IsRowVersion kept**: MassTransit saga has `uint RowVersion`, not `IConcurrencyToken.Version`. The ConcurrencyTokenConvention only handles entities implementing `IConcurrencyToken`, so the saga's IsRowVersion must remain explicit.
- **Addresses ToTable kept**: `UserProfileConfiguration.OwnsMany(Addresses).ToTable("Addresses", "profiles")` specifies a PostgreSQL schema (`profiles`), which is domain-specific and not auto-generated.
- **CategoryName/ProductName HasConversion kept**: These are `readonly record struct` value objects (not `StronglyTypedId<T>`), so the convention doesn't apply.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed StronglyTypedIdConvention: ValueConverter constructor mismatch**
- **Found during:** Task 2 (Generate EF Core migrations)
- **Issue:** `Activator.CreateInstance(typeof(ValueConverter<,>).MakeGenericType(...), lambdaExpr, lambdaExpr)` failed because `ValueConverter<TModel, TProvider>` constructor requires typed `Expression<Func<TModel, TProvider>>` not untyped `LambdaExpression`. Runtime constructor resolution fails silently.
- **Fix:** Introduced `CreateTypedConverter<TId, TUnderlying>()` generic method that creates properly-typed `Expression<Func<TId, TUnderlying>>` and returns `new ValueConverter<TId, TUnderlying>(toProvider, fromProvider)`. Called via `MethodInfo.MakeGenericMethod(stronglyTypedIdType, underlyingType).Invoke(null, null)`.
- **Files modified:** `src/MicroCommerce.ApiService/Common/Persistence/Conventions/StronglyTypedIdConvention.cs`
- **Verification:** Migration generation succeeded for all 8 DbContexts after fix
- **Committed in:** `1d68d9bc` (Task 2 commit)

**2. [Rule 1 - Bug] Fixed BaseDbContext.OnConfiguring incompatibility with DbContext pooling**
- **Found during:** Task 2 (Generate EF Core migrations - design time instantiation)
- **Issue:** `AddNpgsqlDbContext` (Aspire) uses DbContext pooling. Pooled contexts cannot modify `DbContextOptions` in `OnConfiguring`. Migration tool threw: `'OnConfiguring' cannot be used to modify DbContextOptions when DbContext pooling is enabled.`
- **Fix:** Removed `OnConfiguring` override from `BaseDbContext`. Added `options.UseSnakeCaseNamingConvention()` to each `configureDbContextOptions` lambda in `Program.cs` for all 8 DbContext registrations.
- **Files modified:** `BaseDbContext.cs`, `Program.cs`
- **Verification:** All 8 migrations generated successfully; build passes with zero errors
- **Committed in:** `1d68d9bc` (Task 2 commit)
