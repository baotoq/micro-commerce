<objective>
Create all ArgoCD manifests (root app-of-apps, 6 child Applications, NodePort service, TLS-disable ConfigMap) and restructure Kustomize for per-service ArgoCD management.

Purpose: ArgoCD needs Application manifests to know what to manage. The existing Kustomize structure must be adapted so each child Application can independently sync its service. Infrastructure services need sealed-secret.yaml removed from their kustomization.yaml (applied separately by bootstrap) so ArgoCD can sync from Git where those files do not exist.

Output: Complete ArgoCD manifest directory (`infra/k8s/argocd/`), updated kind-config.yaml, restructured Kustomize overlays.
</objective>

         namespace: argocd
         finalizers:
         - resources-finalizer.argocd.argoproj.io

           repoURL: https://github.com/baotoq/micro-commerce.git
           targetRevision: master
           path: infra/k8s/argocd/apps
         destination:
           server: https://kubernetes.default.svc
           namespace: argocd
         syncPolicy:
           automated:
             selfHeal: true
             prune: true
       ```

         name: argocd-server-nodeport
         namespace: argocd
       spec:
         type: NodePort
         selector:
           app.kubernetes.io/name: argocd-server
         ports:
         - name: http
           port: 80
           targetPort: 8080
           nodePort: 30443
       ```
       Use HTTP (port 80 -> targetPort 8080) because TLS will be disabled via ConfigMap patch.

    4. **Create `infra/k8s/argocd/argocd-cmd-params-cm.yaml`**: ConfigMap patch to disable TLS:

         namespace: argocd

       - `syncOptions: [CreateNamespace=true]`

       - `infra/k8s/base/postgres/kustomization.yaml`: Remove `- sealed-secret.yaml`, keep `- statefulset.yaml` and `- service.yaml`
       - `infra/k8s/base/rabbitmq/kustomization.yaml`: Remove `- sealed-secret.yaml`, keep `- deployment.yaml` and `- service.yaml`

       WHY: sealed-secret.yaml files are generated at runtime by bootstrap.sh and do not exist in Git. ArgoCD syncs from Git, so referencing missing files would cause Kustomize build failures. The bootstrap script will apply sealed secrets directly via `kubectl apply -f` instead.

       ```yaml
       apiVersion: kustomize.config.k8s.io/v1beta1
       kind: Kustomization

       namespace: micro-commerce

       resources:
       - ../../../base/apiservice

       images:
       - name: apiservice
         newName: apiservice
         newTag: dev
       ```

       `infra/k8s/overlays/dev/gateway/kustomization.yaml`:
       ```yaml
       apiVersion: kustomize.config.k8s.io/v1beta1
       kind: Kustomization

       namespace: micro-commerce

       resources:
       - ../../../base/gateway

       images:
       - name: gateway
         newName: gateway
         newTag: dev
       ```

       `infra/k8s/overlays/dev/web/kustomization.yaml`:
       ```yaml
       apiVersion: kustomize.config.k8s.io/v1beta1
       kind: Kustomization

       namespace: micro-commerce

       resources:
       - ../../../base/web

       images:
       - name: web
         newName: web
         newTag: dev
       ```

       WHY per-service overlays: Each child ArgoCD Application needs its own Kustomize path. App services need image tag patching via the overlay. Per-service overlays also prepare for Phase 27 where CI will update individual image tags.

    3. **Update `infra/k8s/overlays/dev/kustomization.yaml`** to reference the per-service overlays instead of the monolithic base:

Remove the top-level `images:` block since per-service overlays now handle image tags.
