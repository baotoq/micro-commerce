  <action>
    1. Refactor `GetAllOrdersQueryHandler.Handle()`:
       - Add usings: `using Ardalis.Specification;`, `using Ardalis.Specification.EntityFrameworkCore;`, `using MicroCommerce.ApiService.Features.Ordering.Application.Specifications;`
       - Replace the inline status filtering with spec composition:
         ```
         IQueryable<Order> query = context.Orders.AsNoTracking();

         if (!string.IsNullOrWhiteSpace(request.Status)
             && OrderStatus.TryFromName(request.Status, ignoreCase: true, out OrderStatus? statusFilter))
         {
             query = query.WithSpecification(new OrdersByStatusSpec(statusFilter!));
         }
         ```
       - Keep `OrderByDescending(o => o.CreatedAt)` in handler (sorting is handler concern)
       - Keep Skip/Take pagination in handler
       - Keep the `.Select(o => new OrderSummaryDto(...))` projection in handler unchanged
       - Keep totalCount query structure, applying WithSpecification for consistency

    2. Refactor `GetOrdersByBuyerQueryHandler.Handle()`:
       - Add same usings as above
       - Replace inline buyer + status filtering with spec composition:
         ```
         Specification<Order> spec = new OrdersByBuyerSpec(request.BuyerId);

         if (!string.IsNullOrWhiteSpace(request.Status)
             && OrderStatus.TryFromName(request.Status, ignoreCase: true, out OrderStatus? statusFilter))
         {
             spec = spec.And(new OrdersByStatusSpec(statusFilter!));
         }

         IQueryable<Order> query = context.Orders
             .AsNoTracking()
             .WithSpecification(spec);
         ```
       - This demonstrates And() composition in ordering domain (buyer + optional status)
       - Keep OrderByDescending, Skip/Take, Select projection in handler unchanged

    3. Use explicit types throughout per C# conventions: `IQueryable<Order> query`, `Specification<Order> spec`, `int totalCount`, `List<OrderSummaryDto> items`.

    IMPORTANT: Maintain identical API contract — same OrderListDto/OrderSummaryDto shape, same pagination behavior.
