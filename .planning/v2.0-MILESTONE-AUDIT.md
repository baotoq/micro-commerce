## Cross-Phase Integration

### Connected Flows (7/7)

1. **Entity hierarchy chain** (Phase 15 → 21): `Entity<TId>` → `BaseAggregateRoot<TId>` → `AuditableAggregateRoot<TId>` → 5 aggregates + 6 child entities. Order intentionally uses BaseAggregateRoot (documented decision).

2. **Vogen IDs pipeline** (Phase 16.1 → 16 → 21): 14 Vogen IDs → `VogenEfCoreConverters` → `BaseDbContext.ConfigureConventions()` → all 8 DbContexts → `VogenIdSchemaTransformer` for OpenAPI.

3. **Interceptor pipeline** (Phase 15 → 22): 3 stateless interceptors instantiated as singletons → `AddInterceptors()` on all 8 DbContexts (order: SoftDelete → Concurrency → Audit) → mirrored in test factory → 5 integration tests verify runtime behavior. **Previously broken; fixed by Phase 22.**

4. **Result pattern flow** (Phase 17 → 21): FluentResults → `ResultValidationBehavior` in MediatR pipeline → 4 command handlers return `Result` → `result.ToHttpResult()` in endpoints.

5. **SmartEnum flow** (Phase 18 → 21): OrderStatus/ProductStatus SmartEnums → EF `SmartEnumStringConverter` → JSON `SmartEnumNameConverter` → `SmartEnumSchemaTransformer` for OpenAPI → state transitions in domain methods.

6. **Specification flow** (Phase 19 → 21): Ardalis.Specification → 8 specification classes → `WithSpecification()` in query handlers → chained composition for complex queries.

7. **Test infrastructure** (Phase 20 → 22): Testcontainers → `ApiWebApplicationFactory` (with interceptors mirrored) → `IntegrationTestBase` → 182 tests (38 integration + 144 unit).
