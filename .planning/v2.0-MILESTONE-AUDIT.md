## Critical Gaps

### 1. Interceptors Not Wired to DbContexts (ROOT CAUSE)

**Affects:** ENTITY-02 (unsatisfied), ENTITY-04 (partial), ENTITY-05 (partial), ADOPT-03 (partial)

**Location:** `src/MicroCommerce.ApiService/Program.cs`

**Problem:** Three interceptors are registered in DI via `AddScoped<T>()` (lines 157-159) but are never passed to any `DbContextOptionsBuilder` via `AddInterceptors()`. EF Core does not auto-discover `SaveChangesInterceptor` implementations from DI — they must be explicitly added to each DbContext's options.

**Runtime Impact:**
- `AuditInterceptor`: `CreatedAt`/`UpdatedAt` on 5 aggregates remain `0001-01-01T00:00:00+00:00`
- `ConcurrencyInterceptor`: `Version` on 6 entities stays `0` forever; optimistic concurrency silently disabled
- `SoftDeleteInterceptor`: Hard deletes would not be converted to soft deletes (dormant — no entities implement ISoftDeletable)

**Fix:** Add `AddInterceptors()` calls in each of the 8 `configureDbContextOptions` lambdas in Program.cs:
```csharp
options.AddInterceptors(
    sp.GetRequiredService<AuditInterceptor>(),
    sp.GetRequiredService<ConcurrencyInterceptor>(),
    sp.GetRequiredService<SoftDeleteInterceptor>(),
    sp.GetRequiredService<DomainEventInterceptor>());
```
